# - ошибки
@ - альтернативный сценарий

Составить ежедневный отчёт
1. Пользователь находится в главном окне
2. Пользователь нажимает на составить ежедневный отчёт
-Программа выводит отчёт
-Пользователь нажимает сохранить
-Выводится окно с путём для сохранения
-Пользователь выбирает путь сохранения
-Программа сохраняет отчёт на введённый путь
3. Пользователь находится в главном меню
#Если СУБД не открывается, то выводится окно с сообщением об ошибке
@Если пользователь забыывает составить отчёт, то пользователь может составить отчёт за определённый день
@Если пользователь хочет распечатать, то пользователь может нажать на кнопку печать


Регистрация
1. Пользователь находится в окне регистрации
2. Пользователь вводит свои данные (директор)
-Пользователь нажимает на кнопку сохранить
3. Пользователь находится в главном меню
#Если пользователь вводит неправильные символы, то выводится сообщение с разрешёнными символами в поле
#Если СУБД не открывается, то выводится окно с сообщением об ошибке

Авторизация
1. Пользователь находится в окне авторизации
2. Пользователь вводит логин и пароль
-Пользователь нажимает на кнопку вход
3. Пользователь находится в главном меню
#Если не правильный пароль или логин, то программа выводит сообщение "неправильные пароль или логин" и убирает пароль
#Если пользователь вводит неправильные символы, то выводится сообщение с разрешёнными символами в поле
#Если СУБД не открывается, то выводится окно с сообщением об ошибке
