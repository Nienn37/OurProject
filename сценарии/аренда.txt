Просмотреть аренду
1. Пользователь находится в главном меню
2. Пользователь нажал на кнопку список аренд
3. Пользователь находится в списке аренд
#Если СУБД не открывается, то выводится окно с сообщением об ошибке


Добавить аренду
1. Пользователь находится в главном меню/ в списке аренд
2. 
-Пользователь нажимает добавить аренду
-Открывается форма ввода данных аренды
-Пользователь вводит данные аренды
-Пользователь нажимает сохранить
-Программа добавляет аренду в базу данных
-Форма ввода данных закрывается
3. Пользователь находится в обновленном списке аренд
#Если пользователь ввёл неправильные символы, то выводится сообщение с разрешёнными символами в поле
#Если пользователь не заполнил все обязательные поля, то выводится окно с сообщением "не все обязательные поля заполнены"
#Если аренда с такими данными в БД уже существует, то выводится окно с сообщением "такая аренда уже существует"


Удалить аренду
1. Пользователь находится в списке аренд
2. 
-Пользователь выбрал аренду(ы)
-Пользователь нажал на кнопку удалить
-Программа выводит окно с подтверждением
-Пользователь нажал на кнопку подтвердить
-Программа удаляет аренду из базы данных
3. Пользователь находится в обновленном списке аренд
#Если пользователь случайно удалили не то что нужно, то можно использовать сочетание клавиш ctrl+z.
#Если пользователь нажал на кнопку отменить, то возвращается в список дисков с выбранными дисками


Поиск аренды
1. Пользователь находится в списке аренд
2. Пользователь ввел в поле для поиска данные аренды
3. Пользователь находится в списке найденных аренд
#Если программа ничего не нашла, то выводит сообщение "по вашему запросу ничего не найдено"
#Если пользователь ввёл неправильные символы, то выводится сообщение с разрешёнными символами в поле


Редактировать аренду
1. Пользователь находится в списке аренд
2. 
-Пользователь выбрал аренду
-Пользователь нажал на кнопку редактировать
-Открывается форма ввода данных, уже заполненная данными выбранной аренды
-Пользователь отредактировал нужные данные
-Пользователь нажал на кнопку сохранить
-Программа сохраняет новые данные в базу данных
-Форма ввода данных закрывается
3. Пользователь находится в обновленном списке аренд
#Если пользователь выбрал несколько аренд, то кнопка редактировать будет неактивна (тускнеет)
#Если пользователь ввёл неправильные символы, то выводится сообщение с разрешёнными символами в поле
#Если пользователь не заполнил все обязательные поля, то выводится окно с сообщением "не все обязательные поля заполнены"
#Если аренда с такими данными в БД уже существует, то выводится окно с сообщением "такая аренда уже существует"


Продлить аренду
1. Пользователь находится в списке аренд
2. 
- Пользователь выбрал аренду
- Пользователь нажал на кнопку продлить
- Программа открывает окно со сроком продления
- Пользователь вводит срок продления
- Нажали продлить
3. Пользователь находится в обновленном списке аренд
#Если пользователь выбрал несколько аренд, то кнопка продлить будет неактивна (тускнеет)


Закрыть аренду
1. Пользователь находится в списке аренд
2. 
- Пользователь выбрал аренду
- Пользователь нажал на кнопку закрыть аренду
- Программа закрывает аренду
3. Находимся в обновленном списке аренд
#Если пользователь выбрал несколько аренд, то кнопка закрыть аренду будет неактивна (тускнеет)
