# - ошибки
@ - альтернативный сценарий

Просмотреть черный список
1. Пользователь находится в главном окне
2. Пользователь выбирает раздел черный список
3. Пользователь находится в черном списке клиентов
#Если СУБД не открывается, то выводится окно с сообщением об ошибке

Добавить в черный список
1. Пользователь находится в окне черного списка клиентов
2. 
-Пользователь выбирает клиента(ов)
-Пользователь нажимает на кнопку добавить в черный список 
-Программа добавляет клиента(ов) в черный список 
3. Пользователь находится в обновленном окне черного списка клиентов
#Если клиент уже находится в черном списке, то выводится окно с сообщением в котором выводит клиентов, которые уже находятся в чёрном списке
#Если СУБД не открывается, то выводится окно с сообщением об ошибке


Убрать из черного списка
1. Пользователь находится в окне черного списка клиентов
2. 
-Пользователь выбирает клиента(ов)
-Пользователь нажимает на кнопку убрать из черного списка
-Программа убирает клиента(ов) из черного списка
3. Пользователь находится в обновленном окне черного списка клиентов
#Если пользователь случайно убирает не то что нужно, то можно использовать сочетание клавиш ctrl+z
#Если СУБД не открывается, то выводится окно с сообщением об ошибке

Поиск в черном списке
1. Пользователь находится в окне черного списка клиентов
2. Пользователь вводит в поле для поиска данные клиента
3. Пользователь находится в окне черного списка найденных клиентов
#Если программа ничего не находит, то выводит сообщение "по вашему запросу ничего не найдено"
#Если пользователь вводит неправильные символы, то выводится сообщение с разрешёнными символами в поле
#Если СУБД не открывается, то выводится окно с сообщением об ошибке
